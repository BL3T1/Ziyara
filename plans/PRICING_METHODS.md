# Pricing Methods Report: Ziyarah Platform

This report details the various pricing models implemented across the Ziyarah Modular Monolith to support its multi-vertical hospitality ecosystem.

## 1. Vertical-Specific Pricing Models

The system employs different pricing strategies optimized for each service category.

| Service Category | Pricing Model | Logic | Components |
| :--- | :--- | :--- | :--- |
| **Hotels & Resorts** | **Time-Based (Per Night)** | Base Price × Nights × Rooms | Base Rate, Seasonal Multiplier, Tax |
| **Restaurants** | **Reservation-Based** | Flat Fee or Minimum Spend | Covers reservation lock, applicable to bill |
| **Taxis** | **Dynamic (Distance/Time)** | Base + (Km × Rate) + (Min × Rate) | Vehicle Type, Peak Hour Surcharge |
| **Trips & Tours** | **Unit-Based (Per Person)** | Unit Price × Number of Guests | Adult/Child pricing, Group Discounts |

## 2. Global Pricing Components

Regardless of the vertical, several shared components influence the final transaction amount.

### 2.1 Ziyarah Commission (Dynamic & Stacked)
- **Standard Default**: A **10% default commission** is applied to all services.
- **Dynamic Override**: Super Admins and Managers can set specific percentages for individual providers via the Admin Dashboard.
- **Stacked Discount Logic**: 
    1. Apply **Provider Discount** first.
    2. Apply **Company Discount** second.
    3. Calculate **Commission** on the resulting base.
- **Master Formula**: `Final Price = (Provider_Base - Disc_P - Disc_C) * (1 + Commission_Rate)`.

### 2.2 Dynamic Adjustments
1.  **Seasonal Multipliers**: Managed by providers via the Client Portal. Allows for higher rates during holidays/peak seasons.
2.  **Last-Minute Deals**: Discounted rates for immediate availability (e.g., Taxis off-peak, same-day Hotel bookings).

## 3. Currency & Exchange Rates

The system supports multiple currencies with real-time conversion.
- **Base Currency**: All internal accounting is performed in a consistent base currency (e.g., USD or local treasury currency).
- **Exchange Rate Engine**: Fetches rates via an external API; cached for performance (1-hour TTL).
- **Customer View**: Customers see prices in their `preferred_currency` defined in their profile.

## 4. Discount & Promotion Impact

Pricing is affected by the **Discount Module** state machines.
- **Manual Discounts**: Created by Sales/Finance (subject to approval chains).
- **Promo Codes**: Applied at checkout; reduces the `Total Amount` before payment.
- **Stacking Rules**: System generally allows one promo code per booking unless overridden by a "Global Campaign" flag.

---

## 5. Tax & Surcharge Handling

Each domain module (`hotel_`, `taxi_`, etc.) can define localized taxes.
- **Inclusive Pricing**: By default, prices shown to customers include all mandatory taxes to maintain transparency and trust.
